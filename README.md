# Система Help Desk

## О проекте

Учебный проект по системному анализу, посвященный оптимизации процесса обработки обращений пользователей с помощью внедрения системы Help Desk.

Цель проекта — анализ существующего процесса (As-Is), выявление проблем, разработка целевого процесса (To-Be) и проектирование информационной системы для автоматизации обработки заявок.

## Бизнес-проблема


Текущий процесс обработки обращений через электронную почту приводит к следующим проблемам:

- длительное время первого ответа пользователю;
- длительное назначение исполнителя на заявку;
- потеря части обращений;
- нарушение целевых сроков обработки (SLA).

## Ключевые показатели

### As-Is

| Метрика | Значение |
|----------|----------|
| Время первого ответа | 24 часа |
| Время назначения исполнителя | 2 часа |
| Потерянные заявки | 15% |
| Нарушение SLA | 20% |

### To-Be

| Метрика | Значение |
|----------|----------|
| Время первого ответа | 6 часов |
| Время назначения исполнителя | 10 минут |
| Потерянные заявки | 1% |
| Нарушение SLA | 5% |

## Предлагаемое решение

Внедрение Help Desk системы, обеспечивающей:

- регистрацию обращений;
- автоматическое назначение исполнителей;
- контроль SLA;
- управление статусами заявок;
- уведомления пользователей;
- хранение истории обработки обращений.

## Роли пользователей

### Пользователь

Создает и отслеживает заявки.

### Оператор поддержки 

Обрабатывает обращения пользователей.


### Администратор

Управляет пользователями и настройками системы.

## Основные возможности

- создание заявки;
- просмотр статуса заявки;
- назначение исполнителя;
- изменение статуса заявки;
- добавление комментариев;
- эскалация заявки;
- контроль SLA;
- закрытие заявки.
